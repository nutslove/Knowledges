- 関数名の先頭にアンダースコア`_`を付けるのは、**「プライベート」や「内部使用」** を示す慣習

## 1. シングルアンダースコア `_function_name`
- 「このメソッド/関数は内部実装用で、外部から直接呼び出すべきではない」という**慣習的な目印**
- 技術的にはアクセス可能だけど、「使わないでね」という開発者間の約束事
- `from module import *` でインポートされなくなる
- 例  
  ```python
  class MyClass:
      def public_method(self):
          return self._internal_helper()
      
      def _internal_helper(self):  # 内部用
          return "内部処理"
  ```

## 2. ダブルアンダースコア `__function_name`
- 名前マングリング（Name Mangling） が適用される
- Pythonが自動的に名前を `_ClassName__function_name` に変換し、サブクラスでの名前衝突を防ぐ
- 例  
  ```python
  class MyClass:
      def __private_method(self):
          return "これはプライベートメソッド"
  
  obj = MyClass()
  # obj.__private_method()  # ❌ AttributeError
  print(obj._MyClass__private_method())  # ✅ 正常に呼び出せる
  ```

  ```python
  class BaseEmployee:
      def __init__(self, name: str):
          self.name = name

      def __get_regular_hours(self):
          """基底クラスの労働時間計算"""
          return 160  # 月160時間

      def calculate_base_pay(self, hourly_rate: int):
          hours = self.__get_regular_hours()  # _BaseEmployee__get_regular_hours を呼ぶ
          return hours * hourly_rate


  class PartTimeEmployee(BaseEmployee):
      def __get_regular_hours(self):
          """パートタイム用の計算（親のメソッドを上書きしない！）"""
          return 80  # 月80時間

      def calculate_part_time_pay(self, hourly_rate: int):
          hours = self.__get_regular_hours()  # _PartTimeEmployee__get_regular_hours を呼ぶ
          return hours * hourly_rate


  if __name__ == "__main__":
      part_timer = PartTimeEmployee("Suzuki")

      # 親クラスのメソッドは親の __get_regular_hours (160時間) を使う
      base_pay = part_timer.calculate_base_pay(1000)
      print(f"基本給計算: {base_pay}円")  # 160,000円

      # 子クラスのメソッドは子の __get_regular_hours (80時間) を使う
      part_time_pay = part_timer.calculate_part_time_pay(1000)
      print(f"パート給計算: {part_time_pay}円")  # 80,000円
  ```
